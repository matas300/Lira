# Scadenze (v1) — Spec + Implementation Plan

> Eseguire task-by-task in TDD (per le funzioni pure), commit frequenti.

**Goal:** Pagina "Scadenze" (`/scadenze`) che mostra le scadenze fiscali dell'anno selezionato (14 righe del scadenziario), con stato pagamento, e azioni "Segna pagato" / "Annulla". Anno dal selettore sidebar. Solo frontend: il backend `scadenziario` + `pagamenti` esiste già.

**Architecture:** Pagina vanilla TS che consuma `GET /api/scadenziario/:year`. Funzioni di render PURE (testabili senza DOM) + `mount` che fa fetch e cabla le azioni di pagamento via `POST /api/pagamenti/quick-pay` e `DELETE /api/pagamenti/:id`. Chip "timing" calcolato client-side da `dueDate` vs oggi.

**Tech Stack:** vanilla TS + Vite, `lib/api.ts`, `lib/dom` (`mountPage`, `esc`), `lib/year` (`getYear`), `node --test`.

## Contratti API (dal backend già esistente)

**`GET /api/scadenziario/:year`** (auth session; profilo da sessione). 404 `YEAR_SETTINGS_NOT_FOUND` se settings mancanti.
Output `ScadenziarioView`:
```
{
  year, method: 'storico'|'previsionale',
  rows: ScadenziarioRow[14],
  summary: { totalDue, totalPaid, totalResidual, nextDue: ScadenziarioRow|null },
  warnings: { code, severity, title, message }[],
  methodComparison, transition, rulesRef
}
```
`ScadenziarioRow`:
```
{
  id,                       // scheduleKey es. "imposta_saldo_2025"
  title, family, kind: 'tax'|'contribution',
  competenceYear,
  dueDate,                  // ISO YYYY-MM-DD (effettiva)
  dueDateOriginal, dueDateRolled, prorogaApplied,
  amount: { low, high, point },
  certainty: 'official'|'estimated'|'forecast',
  payments: { id, data, importo, mode:'pure'|'mixed' }[],
  paidTotal,
  status: { code:'paid'|'underpaid'|'overpaid'|'estimated'|'to_confirm', label, tone:'ok'|'warn'|'danger'|'info' },
  explanation
}
```
"Pagata/archiviata" ⇒ `status.code === 'paid'`. Residuo riga = `amount.point − paidTotal` (≥0).

**`POST /api/pagamenti/quick-pay`** body `{ scheduleKey, importo, data?, tipo? }` → PagamentoPublic. (importo richiesto.)
**`DELETE /api/pagamenti/:id`** → `{ ok:true }`. Per "annullare" una riga: DELETE su ogni `row.payments[].id`.

### Task 1: Helper timing (`lib/scadenza-timing.ts`)

**Files:** Create `src/client/lib/scadenza-timing.ts`; Test `scadenza-timing.test.ts`

`scadenzaTiming(dueDateIso, today)` → `{ state:'scaduta'|'imminente'|'futura', label, tone:'danger'|'warn'|'ok' }`.
- scaduta: dueDate < today → tone danger, label "Scaduta"
- imminente: 0 ≤ (dueDate − today) ≤ 30 giorni → tone warn, label "Tra Ng" o "Oggi"
- futura: > 30 giorni → tone ok, label data
`today` è un parametro (ISO o Date) per testabilità; in app passare la data odierna.

- [ ] **Step 1: failing test** — casi: dueDate ieri→scaduta; oggi→imminente "Oggi"; +10g→imminente; +60g→futura. (Passare `today` fisso, es. '2026-06-16'.)
- [ ] **Step 2: run fail** `npx tsx --test src/client/lib/scadenza-timing.test.ts`
- [ ] **Step 3: implementare** (parse ISO → giorni di differenza; nessun uso di `Date.now` interno, usa `today`).
- [ ] **Step 4: run pass**
